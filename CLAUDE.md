# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с данным репозиторием.

## Что это за репозиторий

Личная **база знаний, поддерживаемая LLM** (паттерн LLM Wiki от Karpathy). Нет системы сборки, тестов и деплоя — только структурированная коллекция Markdown-файлов, которую ведёт LLM-агент по поручению пользователя.

Полные правила воркфлоу — в `.schema/CLAUDE.md`. Читать в начале каждой сессии.

## Трёхслойная структура

| Слой | Путь | Правило |
|------|------|---------|
| Исходники | `raw/` | **Никогда не изменять.** Пользователь кладёт сюда файлы; LLM только читает. |
| Вики-страницы | `wiki/` | LLM создаёт и обновляет. |
| Схема | `.schema/CLAUDE.md` | Конфигурация воркфлоу — только для чтения. |

## Структура wiki/

```
wiki/
├── index.md       # Каталог всех страниц — обновлять при каждом изменении
├── log.md         # Лог операций — только дополнение (append-only)
├── concepts/      # Концепции и темы
├── entities/      # Люди, организации, проекты
├── sources/       # Саммари каждого обработанного источника
└── queries/       # Сохранённые ответы на вопросы
```

## Приоритет источников

**Всегда сначала искать ответ в локальной базе знаний (`wiki/`), и только если информации недостаточно — обращаться к внешним источникам (интернет, MCP-инструменты и др.).**

1. Прочитать `wiki/index.md` и найти релевантные страницы.
2. Прочитать найденные страницы и проверить, достаточно ли информации для ответа.
3. Только если локальная база не покрывает вопрос — использовать внешние источники.
4. Если внешний источник дал ценную новую информацию — предложить пользователю затянуть её в `wiki/`.

## Основные воркфлоу

**Ingest** (`raw/` → `wiki/`): прочитать источник → обсудить с пользователем акценты → создать страницу в `wiki/sources/` → обновить/создать страницы концепций и сущностей → обновить `wiki/index.md` → дописать запись в `wiki/log.md`. Один источник затрагивает обычно 5–15 страниц.

**Query**: прочитать `wiki/index.md` → прочитать релевантные страницы → синтезировать ответ с цитатами через `[[wikilinks]]` → при необходимости сохранить в `wiki/queries/`.

**Lint**: пройти по вики в поисках противоречий, устаревших утверждений, осиротевших страниц, сломанных ссылок → исправить на месте → дописать запись в `wiki/log.md`.

## Формат страниц

Каждая вики-страница должна содержать YAML frontmatter:

```yaml
---
title: "Название страницы"
tags: [тег1, тег2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: [source-file.md]   # опционально
---
```

- Перекрёстные ссылки — через `[[wikilinks]]` (Obsidian-совместимый Markdown).
- Много маленьких связанных страниц лучше одной большой.
- Язык: заголовки страниц и названия концепций — английский; тело страницы — русский. Код и технические термины остаются на английском.

## Дисциплина index.md и log.md

- `index.md` обновлять **каждый раз**, когда вики-страница добавляется, изменяется или удаляется.
- `log.md` — только дополнение, старые записи не редактировать. Формат заголовка: `## [YYYY-MM-DD] <операция> | <название>`, где операция — одно из `ingest`, `query`, `lint`, `create`, `update`.

## Интеграция с Obsidian

Директория `wiki/` открывается как Obsidian vault. Поддерживается Graph view, Dataview (запросы по YAML frontmatter) и Marp (генерация слайдов из вики-страниц).
